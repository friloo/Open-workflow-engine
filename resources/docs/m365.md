# Microsoft 365 — SSO und Benutzer-Sync

## SSO-Login

Konfiguration unter *Verwaltung -> Systemeinstellungen -> Microsoft 365*.

- Client-ID, Tenant-ID, Client-Secret (verschluesselt gespeichert)
- Redirect-URI: muss exakt in der Azure-AD-App-Registration eingetragen sein
- Auto-Provisioning: legt unbekannte Benutzer automatisch mit Standardrolle an
- Standardrolle waehlbar

Sobald aktiviert, erscheint auf der Login-Seite "Mit Microsoft anmelden".

## Benutzer-Sync

Voraussetzung: in Azure AD die Application-Permission `User.Read.All`
mit Admin Consent vergeben.

- Manueller Trigger: Button **Sync jetzt ausfuehren**
- Per Cron: `php artisan m365:sync-users`
- **Test-Verbindung** prueft Credentials ohne Daten zu importieren

Importiert werden: `displayName`, `mail`, `jobTitle`, `department`,
`mobilePhone`, `manager`. Der Manager wird auf den `supervisor_id`-FK
gemappt.
